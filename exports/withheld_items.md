# Withheld Items

This prototype export bundle does not attempt to publish every repository object as planner-safe knowledge.

## Typical reasons for withholding

- the item is still heavily constrained by unresolved unknowns
- the item is useful as research context but not yet safe as a planner default
- the item still depends too strongly on unresolved live verification even when canonical source derivatives are present
- the item is a contextual strategy pattern rather than a universal rule

## Current examples

- `M-0008`
  Body-economy modifier behavior is still constrained by unresolved stacking and Terraforming questions.
- `CS-0001`
  The Industrial Anchor plus High Tech Support Body pattern is promising, but still depends on unresolved commodity-mapping and rank-order questions before it should be treated as a generic planner-safe default.

## Rule

Withholding an item from a release does not mean deleting it from the repository.
It means the repository is keeping the research visible while refusing to overstate certainty.
