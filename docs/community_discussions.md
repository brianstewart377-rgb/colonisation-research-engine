# Community discussions

GitHub Discussions should eventually become part of the research process.

## Intended use

Discussions are for:

- proposing possible new mechanics
- surfacing contradictory evidence
- sharing experiment outcomes
- debating interpretation before work is formalized

## Examples

- “I think I have found a new mechanic.”
- “This contradicts Mechanic M-014.”
- “Patch 4.x seems to have changed the expected outcome.”

## Separation from issues

- `Discussions` are for research conversation and evolving interpretation
- `Issues` are for concrete work items such as ontology changes, schema design, or dashboard tasks

## Current state

If Discussions are not enabled yet, this document still defines the intended governance so the repository can adopt them later without confusion.
